# 离线文本标注复核工具

完全在本地运行的文本标注复核系统，不依赖任何外部 AI 服务。支持多角色权限管理、标注冲突检测、复核任务分配、标签方案版本化、完整审计追溯。

---

## 功能特性

### 🔐 三角色权限体系
| 角色 | 权限 |
|------|------|
| **管理员 (admin)** | 全部功能：配置标签方案、导入样本/标注、检测冲突、分配复核、用户管理、导出证据、查看审计历史 |
| **标注员 (annotator)** | 导入样本、导入自己的标注、查看样本和标注（只能看到自己的标注） |
| **复核员 (reviewer)** | 查看和复核分配给自己的任务、查看冲突详情 |

### 📑 核心流程
1. **配置标签** - 创建/升级标签方案，支持多版本共存
2. **导入样例** - CSV 批量导入，自动检测重复样本编号
3. **导入标注** - 多人标注结果导入，自动检测未知标签
4. **发现冲突** - 同一样本多标注员标签不一致自动检测
5. **分配复核** - 管理员分配冲突给复核员（**复核员不能复核自己参与标注的样本**）
6. **双方意见** - 复核页面对比查看各方标注意见和理由
7. **确认标签** - 复核员选择最终标签并填写复核意见
8. **导出证据** - CSV/JSON 导出完整证据链

### 🛡️ 关键约束处理
- **未知标签**：标注中的标签不在方案中时标记为"未知标签"，不丢失数据
- **重复样本编号**：导入时自动检测并跳过重复编号，不覆盖原数据
- **复核员回避**：参与某样本标注的复核员不能被分配该样本的复核任务
- **标签方案版本隔离**：方案升级创建新版本，**旧版本所有数据独立保存，不自动迁移到新版本**

### 📜 可追溯性
- 所有操作（登录、创建、修改、分配、复核、导出）均写入修订历史
- 标签方案多版本永久保存
- 冲突从检测到分配到解决的完整链路可查
- 复核员决策、意见、时间戳完整留存
- 数据重启后全部保留（SQLite 本地文件）

## 快速开始

### 环境要求
- Python 3.8+
- Windows / macOS / Linux

### 安装与运行

```bash
# 1. 进入项目目录
cd lfc-00028

# 2. 安装依赖
pip install -r requirements.txt

# 3. 初始化样例数据（推荐，用于体验完整功能）
python seed_data.py

# 4. 启动服务
python app.py
```

启动后浏览器访问：**http://127.0.0.1:5000**

## 权限边界验证建议

运行 `python seed_data.py` 后可按以下步骤验证各功能：

### 1. 标注员权限验证（annotator1 / anno123）
- ✅ 登录后可见：仪表盘、样本管理、标注结果
- ❌ 不可见：冲突管理分配、复核任务、用户管理、标签方案、数据导出、修订历史
- ✅ 标注结果页只能看到 annotator1 自己的标注
- ❌ 导入标注时不能选择其他标注员

### 2. 复核员权限验证（reviewer1 / review123）
- ✅ 可见：仪表盘、样本、标注、冲突、复核任务
- ❌ 不可见：用户管理、标签方案管理、导出、修订历史
- ✅ 复核任务只显示分配给自己的任务
- ❌ 不能复核不是分配给自己的任务（直接访问 URL 也会被拦截）

### 3. 管理员权限验证（admin / admin123）

#### 3.1 冲突检测与分配
1. 进入 **冲突管理** → 选择"情感分析"方案 → 点击"检测冲突"
2. 应检测出约 5-6 个冲突（标注员甲乙对同一样本标签不一致）
3. 进入冲突详情 → 分配复核员给 reviewer1
4. ⚠️ **验证回避机制**：标注员乙 annotator2 同时也是复核员 reviewer2？不是的话可以尝试创建一个既是标注员又是复核员的用户测试

#### 3.2 未知标签验证
1. 进入 **标注结果** → 筛选"仅显示未知标签"
2. 应看到 S004、S012、S016 等样本标注了方案中不存在的标签（very_negative、unknown_label）

#### 3.3 标签方案版本隔离验证
1. 进入 **标签方案**，应看到"情感分析"有 v1 和 v2 两个版本
2. v2 多了一个 `very_negative`（强烈负面）标签
3. 点击 v1 查看，**v1 的标注数据仍然独立显示，不会迁移到 v2**
4. 这证明旧结论没有被悄悄迁移到新方案

#### 3.4 复核流程验证
1. 退出 admin，以 **reviewer1 / review123** 登录
2. 进入 **复核任务** → 点击"去复核"
3. 可对比查看标注员甲乙的意见 → 选择最终标签 → 填写意见 → 提交
4. 重新以 admin 登录，在冲突详情中可看到最终结论和完整历史

#### 3.5 导出与追溯
1. admin 进入 **数据导出** → 选择 v1 方案 → 导出 CSV 和 JSON
2. 检查导出文件包含：标签定义、所有标注、冲突状态、最终结论、修订历史
3. 进入 **修订历史**，可看到所有操作的时间、操作人、变更内容

## 数据存储

所有数据保存在本地 SQLite 文件 `data/app.db`，包含以下表：

| 表名 | 说明 |
|------|------|
| `users` | 用户账号（密码加盐哈希存储） |
| `label_schemes` | 标签方案（多版本） |
| `labels` | 标签定义（归属于某方案某版本） |
| `samples` | 文本样本 |
| `annotations` | 标注结果（含未知标签标记） |
| `conflicts` | 冲突记录及最终结论 |
| `conflict_annotations` | 冲突关联的各方标注 |
| `review_tasks` | 复核任务分配与决策 |
| `revision_history` | 全量修订历史审计 |

**完全离线，无任何外部网络请求。**
